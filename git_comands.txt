
git status - статус файлов проэкта

git add - подготовка файлов для комита

git commit -m "описание комита"
git commiy -a -m "описание комита" = git add . и git commit -m "описание комита" в однй команде. Добавляет только модифицированные файлы
git commit --amend -m - дополняет последний комит(новый не создается)

git diff - разница между последним комитом и текущим неотслеживаемым состоянием проэкта
git diff --staged - разница между последним комитом и текущим отслеживаемым состоянием проэкта
git diff commit_id - разница между указаным комитом и текущим состоянием проэкта

git reset [--soft | --mixed | --hard] commit

git checkout [HEAD^ | HEAD~n | commit_id] перемещение к указаному комиту
git checkout (название ветки) - возврат к последнему комиту
git checkout [commit] [--files] - возвращает файлы к состоянию указаннго комита (изменяет файлы)

git clean - удаляет неотслеживаемые файлы
git clean -n - показывает какие файлы будут удалены
git clean -f - удаляет неотслеживаемые файлы


Работа с удаленными репозиториями

git remote -v - список существующих удаленных репозиториев
git remote add название_репозитория адрес_репозитория - добавить новый репозиторий
git remote remove - название_репозитория - удалить репозиторий по названию

git push название_репозитория ветка - отправка локального репозитория на удаленный

git pull - получение обновлений с удаленного репозитория (УР)
